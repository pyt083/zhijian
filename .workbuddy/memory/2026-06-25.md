## 视频播放和缩略图比例修复
- **问题1：视频无法播放**：视频文件名含中文、括号、空格等特殊字符，浏览器 file:// 协议无法直接加载
  - 修复：在 openCaseDetail 函数中，对视频 src 和图片 src 使用 `encodeURI()` 编码
  - 同时将 `<video>` 的 `preload="metadata"` 改为 `preload="auto"`，添加 `style="width:100%;max-height:80vh"`
- **问题2：缩略图比例不匹配**：原 CSS `.case-card-thumb` 强制 `padding-top: 56.25%`（16:9横屏），但 14/18 个视频是 9:16竖屏
  - 修复：移除 CSS 固定 padding-top，改为每个画廊卡片用 inline `style="padding-top: X%"` 动态匹配视频原始比例
  - 竖屏视频 padding-top ≈ 177.78%，横屏 ≈ 56.25%，非标准比例精确计算
  - caseData 新增 `videoWidth` 和 `videoHeight` 字段（18/18 已验证）
- **CSS 优化**：gallery grid 添加 `align-items: start` 使不同高度的卡片顶部对齐

## 仓库迁移到 zhijian + 视频压缩部署（修复视频无法播放）
- **根因**：之前后台推送 1.1GB 视频到 GitHub 失败，远程仓库 0 个视频文件
- **解决方案**：用 ffmpeg 压缩全部 54 个视频
  - ffmpeg 来源：pip install imageio-ffmpeg（自带 ffmpeg 7.1 静态二进制，支持 libx264）
  - 压缩参数：libx264 + preset medium + CRF 30 + scale max 1280px + AAC 96kbps + faststart
  - v18 系列踩坑：分辨率 1080x1906 导致 scale 后产生奇数像素，libx264 报错。修复：链式 scale 滤镜 `scale=...:force_original_aspect_ratio=decrease,scale='trunc(iw/2)*2':'trunc(ih/2)*2'`
  - 压缩结果：1GB → 124MB（8 倍压缩率）
- **仓库**：pyt083/zhijian（从 zhiying-manual 迁移）
- **部署**：index.html 作为入口（GitHub Pages 需要），zhiying-user-manual.html 为源文件
